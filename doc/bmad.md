# BMAD


* installation : https://docs.bmad-method.org/how-to/non-interactive-installation/

## Quickstart

* install
```
./aistack bmad install
./aistack bmad register bash
```

* use in a new shell session
```
bmad help install
bmad install --directory $HOME/workspace/project --modules bmm --tools gemini --yes

bmad install --directory $HOME/workspace/game1 --modules bmm,bmgd --tools gemini --yes


>
bmad BH
bmad-help What is next step ?
```


## Some information

* BMAD Modules available
  * BMad Method (BMM)	Core framework with 34+ workflows
  * BMad Builder (BMB)	Create custom BMad agents and workflows
  * Test Architect (TEA)	Risk-based test strategy and automation
  * Game Dev Studio (BMGD)	Game development workflows (Unity, Unreal, Godot) https://github.com/bmad-code-org/bmad-module-game-dev-studio https://game-dev-studio-docs.bmad-method.org/tutorials/
  * Creative Intelligence Suite (CIS)	Innovation, brainstorming, design thinking



* Agents installed with bmm module
  * https://docs.bmad-method.org/reference/agents/
  * Each agent is available as a skill, generated by the installer. The skill ID (e.g., bmad-dev) is used to invoke the agent. 
  * Triggers are the short menu codes (e.g., CP) and fuzzy matches shown in each agent menu.
  * QA test generation is handled by the bmad-qa-generate-e2e-tests workflow skill, available through the Developer agent. The full Test Architect (TEA) lives in its own module.
